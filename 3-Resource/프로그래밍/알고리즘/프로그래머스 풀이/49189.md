문제를 읽고 다익스트라를 사용하면 풀 수 있다고 쉽게 캐치했다.

하지만 몇몇 테스트 케이스에서 타임 아웃이 발생했는데, q에서 무시할 데이터의 조건을 잘못 썼기 때문이다.

```java
if distances[now] < distance:
	continue
```

처음에 이렇게 조건을 작성했는데, 같은 경우도 이미 최단 경로로 등록된 노드기 때문에 굳이 큐에 넣는 작업이 필요없다.

```java
if distances[now] <= distance:
	continue
```
